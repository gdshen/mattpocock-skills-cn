# 工程

我日常写代码时使用的 skills。

- **[diagnose](./diagnose/SKILL.md)** — 针对疑难 bug 和性能回退的纪律化诊断循环：复现 → 最小化 → 假设 → 加探针 → 修复 → 回归测试。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 根据既有领域模型拷问计划、打磨术语，并内联更新 `CONTEXT.md` 和 ADR。
- **[triage](./triage/SKILL.md)** — 通过 triage roles 的状态机分诊 issues。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 根据 `CONTEXT.md` 里的领域语言和 `docs/adr/` 里的决策，在代码库中寻找 deepening opportunities。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 为 repo 搭建其他工程 skills 会读取的配置：issue tracker、triage label 词汇、domain docs 布局。
- **[tdd](./tdd/SKILL.md)** — 使用 red-green-refactor 循环做测试驱动开发。一次构建一个垂直切片来实现功能或修复 bug。
- **[to-issues](./to-issues/SKILL.md)** — 使用垂直切片把任意 plan、spec 或 PRD 拆成可独立领取的 GitHub issues。
- **[to-prd](./to-prd/SKILL.md)** — 把当前对话上下文转换成 PRD，并作为 GitHub issue 提交。
- **[zoom-out](./zoom-out/SKILL.md)** — 让 agent 抽高一层，对不熟悉的代码区域给出更广的上下文或高层视角。
- **[prototype](./prototype/SKILL.md)** — 构建一次性 prototype 来逼出设计问题：可以是用于 state/business-logic 问题的可运行终端应用，也可以是同一路由上可切换的多个差异很大的 UI 变体。
